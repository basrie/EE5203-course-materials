What is Assembly Language?

The Next Step Down

  • Assembly Language: A low-level programming language.
  • Uses mnemonics (short codes) to represent machine code instructions.
  • Each assembly instruction typically corresponds to one machine instruction.
  • Specific to a particular processor architecture (e.g., ARM, x86).

Why Understand Assembly?

The Deeper Dive

  • Debugging: Essential for understanding complex bugs, especially in embedded systems.
    • When C code doesn’t behave as expected.
    • Analyzing crashes or unexpected behavior.
  • Optimization: Fine-tune code for performance or memory footprint.
    • Identify bottlenecks.
    • Write highly optimized routines (e.g., for critical timing).

More Reasons to Learn Assembly

System-Level Understanding

  • Hardware Interaction: Directly manipulate CPU registers and peripherals.
  • Reverse Engineering: Analyze existing binaries.
  • Security: Understand vulnerabilities and exploits.
  • Boot-up Process: Initializing the microcontroller from reset.
  • Compiler Behavior: See how your C code is translated.

RISC vs. CISC: An Overview

Two Philosophies of CPU Design

  • RISC (Reduced Instruction Set Computer):
    • Simple, fixed-length instructions.
    • Few addressing modes.
    • Emphasis on software complexity (compiler does more work).
    • ARM is a RISC architecture.
  • CISC (Complex Instruction Set Computer):
    • Complex, variable-length instructions.
    • Many addressing modes.
    • Emphasis on hardware complexity (CPU does more work).
    • Example: Intel x86 processors.

RISC Characteristics (ARM)

Simplicity and Speed

  • Simple Instructions: Each instruction performs a single, basic operation.
  • Fixed-Length Instructions: Easier for the CPU to fetch and decode.
  • Load/Store Architecture: Only specific instructions access memory.
  • Many General-Purpose Registers: Reduces memory access, improves speed.
  • Pipelining: Instructions can be processed in stages, improving throughput.

ARM Instruction Set: High-Level Look

What the Processor Can Do

  • The ARM instruction set is the complete list of commands the ARM processor understands.
  • We’ll focus on the main categories:
    • Data Transfer Instructions
    • Data Processing Instructions
    • Branch Instructions

  • Simplified diagram of the ARM instruction set categories.

Data Transfer Instructions

Moving Data Around

  • These instructions move data between registers, or between registers and memory.
  • Load/Store Architecture: A key RISC principle.
    • Data must be loaded into registers before processing.
    • Results must be stored back to memory from registers.

Common Instructions

  • LDR (Load Register): Copies data from memory to a register.
  • STR (Store Register): Copies data from a register to memory.
  • MOV (Move): Copies data from one register to another, or an immediate value to a register.

LDR and STR Examples

Memory Access

; LDR R0, [R1]   ; Load value from memory address in R1 into R0
; STR R0, [R2]   ; Store value from R0 into memory address in R2
; LDR R0, [R1, #4] ; Load value from address (R1 + 4) into R0
; STR R0, [R1, #8]! ; Store value from R0 into address (R1 + 8), then update R1

MOV Example

Register to Register / Immediate to Register

; MOV R0, R1     ; Copy value from R1 to R0
; MOV R0, #10    ; Move immediate value 10 into R0
; MOV R0, #0xFF  ; Move hexadecimal value 255 into R0

Data Processing Instructions

Performing Calculations and Logic

  • These instructions perform arithmetic, logical, and shift operations.
  • They typically operate on values held in registers.
  • Results are usually stored back into a register.

Categories

  • Arithmetic Instructions
  • Logical Instructions
  • Shift Instructions

Arithmetic Instructions

Basic Math Operations

  • ADD: Addition
  • SUB: Subtraction
  • MUL: Multiplication
  • SDIV: Signed Division
  • UDIV: Unsigned Division

Examples

; ADD R0, R1, R2  ; R0 = R1 + R2
; SUB R3, R4, #5  ; R3 = R4 - 5
; MUL R0, R1, R2  ; R0 = R1 * R2

Logical Instructions

Bitwise Operations

  • AND: Bitwise AND
  • ORR: Bitwise OR
  • EOR: Bitwise XOR
  • BIC: Bit Clear (AND with NOT)
  • MVN: Move Not (Bitwise NOT)

Examples

; AND R0, R0, #0xFF ; Mask lower 8 bits of R0
; ORR R1, R1, #0x01 ; Set bit 0 of R1
; EOR R2, R2, #0x80 ; Toggle bit 7 of R2

Shift Instructions

Manipulating Bits

  • LSL (Logical Shift Left): Shifts bits left, fills with zeros.
  • LSR (Logical Shift Right): Shifts bits right, fills with zeros.
  • ASR (Arithmetic Shift Right): Shifts bits right, preserves sign bit.
  • ROR (Rotate Right): Rotates bits, including carry flag.

Examples

; LSL R0, R0, #2  ; R0 = R0 * 4
; LSR R1, R1, #1  ; R1 = R1 / 2 (unsigned)

Branch Instructions

Changing the Flow of Execution

  • These instructions alter the Program Counter (PC) to jump to a different part of the code.

Types

  • Unconditional Branches: Always jump.
  • Conditional Branches: Jump only if certain conditions (flags in APSR) are met.
  • Branch with Link: Used for function calls.

Unconditional Branches

Always Jump

  • B (Branch): Jumps to a specified label or address.

Example

; Loop_Start:
;   ... code ...
;   B Loop_Start  ; Jump back to Loop_Start

Conditional Branches

Jumping Based on Conditions

  • These instructions check the condition flags (N, Z, C, V) in the APSR.
  • Only branch if the condition is true.

Common Conditional Branches

  • BEQ: Branch if Equal (Z flag set)
  • BNE: Branch if Not Equal (Z flag clear)
  • BGT: Branch if Greater Than (signed)
  • BLT: Branch if Less Than (signed)
  • BCS / BHS: Branch if Carry Set / Higher or Same (unsigned)
  • BCC / BLO: Branch if Carry Clear / Lower (unsigned)

Example

; CMP R0, #0      ; Compare R0 with 0, sets Z flag if R0 == 0
; BEQ Label_Zero  ; Branch to Label_Zero if R0 was 0

Branch with Link (BL)

Function Calls

  • BL (Branch with Link): Used to call subroutines (functions).
  • It performs a branch to the target address.
  • Crucially, it saves the address of the next instruction into the Link Register (LR).
  • The subroutine can then return by branching to the address in LR.

Example

; BL MyFunction   ; Call MyFunction, save return address in LR
; ...             ; Execution continues here after MyFunction returns

; MyFunction:
;   ... code ...
;   BX LR         ; Return to caller (branch to address in LR)

How C Code Maps to Assembly

The Compiler’s Role

  • The C compiler is responsible for translating your high-level C code into low-level assembly instructions.
  • This translation is complex and involves many optimizations.
  • Understanding this mapping helps in debugging and optimizing embedded code.

Visual Suggestion

  • Diagram: A C code snippet on one side, with arrows pointing to corresponding assembly instructions on the other side.

C to Assembly Example: Simple Addition

C Code

int a = 5;
int b = 10;
int sum = a + b;

Possible Assembly Output

; MOV R0, #5      ; Load immediate value 5 into R0 (for variable 'a')
; MOV R1, #10     ; Load immediate value 10 into R1 (for variable 'b')
; ADD R2, R0, R1  ; Add R0 and R1, store result in R2 (for variable 'sum')

C to Assembly Example: if Statement

C Code

if (value > 10) {
    result = 1;
} else {
    result = 0;
}

Possible Assembly Output

; LDR R0, [SP, #0]  ; Load 'value' from stack into R0
; CMP R0, #10       ; Compare R0 with 10
; BLE Else_Label    ; Branch if Less than or Equal to (value <= 10)
; MOV R1, #1        ; result = 1
; B End_If          ; Unconditional branch to End_If
; Else_Label:
;   MOV R1, #0      ; result = 0
; End_If:
; STR R1, [SP, #4]  ; Store 'result' back to stack

C to Assembly Example: Function Call

C Code

int main() {
    int x = myFunction(5);
    return 0;
}

int myFunction(int param) {
    return param * 2;
}

Possible Assembly Output (Simplified)

; main:
;   MOV R0, #5      ; Pass 5 as argument to myFunction (R0 is argument register)
;   BL myFunction   ; Call myFunction, save return address in LR
;   MOV R1, R0      ; Store return value (in R0) to x (in R1)
;   MOV R0, #0      ; Return 0 from main
;   BX LR           ; Return from main

; myFunction:
;   LSL R0, R0, #1  ; param * 2 (R0 = R0 << 1)
;   BX LR           ; Return from myFunction
