Conceptual Background: Analog-to-Digital Conversion (ADC)

Bridging the Gap

  • Analog Signals: Continuous, real-world phenomena (e.g., temperature, light intensity).
  • Digital Signals: Discrete values (0s and 1s) understood by microcontrollers.
  • ADC Role: Converts analog voltage levels into digital numbers.

TASK 1: Potentiometer-Controlled Servo Motor

Task 1: Wiring Diagram

Potentiometer (Analog Input)

  • Outer pins → 3.3V and GND
  • Middle (wiper) → PA0 (ADC1_IN0)

Servo Motor (PWM Output)

  • Red wire → 5V
  • Brown/Black → GND
  • Orange/Yellow → PA6 (TIM3_CH1)

Key Points

  • Servo requires 50Hz PWM (20ms period)
  • Pulse width: 0.5ms = 0°, 1.5ms = 90°, 2.5ms = 180°
  • Potentiometer provides 0-3.3V analog signal

Task 1: Code Blueprint - Key HAL Functions

Servo Motor Control Functions

  • ADC (Polled Mode):
    • HAL_ADC_Init(): Initialize ADC peripheral.
    • HAL_ADC_ConfigChannel(): Configure ADC channel for potentiometer.
    • HAL_ADC_Start(): Start ADC conversion.
    • HAL_ADC_PollForConversion(): Wait for conversion completion.
    • HAL_ADC_GetValue(): Read converted digital value.
  • PWM for Servo:
    • HAL_TIM_PWM_Init(): Initialize timer for PWM (50Hz for servo).
    • HAL_TIM_PWM_Start(): Start PWM generation.
    • __HAL_TIM_SET_COMPARE(): Set pulse width (0.5-2.5ms for servo position).

Why these libraries?

  • <stdio.h>: Required for sprintf() (String Print Formatted). It formats data (like integers) into a string buffer.
  • <string.h>: Required for strlen() (String Length). It calculates the exact length of the string for UART transmission.

How sprintf works: sprintf(buffer, "Val: %d", var); prints formatted text into a character array (buffer) instead of the screen.

Task 1: Code Implementation (Main Loop)

Step 7: The While Loop

/* USER CODE BEGIN WHILE */
while (1) {
  HAL_ADC_PollForConversion(&hadc1, HAL_MAX_DELAY);
  uint16_t pot_value = HAL_ADC_GetValue(&hadc1);
  
  // Map ADC (0-4095) to servo pulse (500-2500 = 0°-180°)
  uint32_t servo_pulse = 500 + ((pot_value * 2000) / 4095);
  __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_1, servo_pulse);

  // Send Potentiometer Data to UART
  char msg[50];
  // Calculate Voltage in mV: (ADC_Value * 3300) / 4095
  uint16_t voltage_mv = (pot_value * 3300) / 4095;
  sprintf(msg, "Pot ADC: %hu, Voltage: %hu mV\r\n", pot_value, voltage_mv);
  HAL_UART_Transmit(&huart2, (uint8_t*)msg, strlen(msg), 100);
  
  HAL_Delay(20);
/* USER CODE END WHILE */
}

TASK 2: LDR-Controlled LED Brightness (Interrupt-Driven ADC)

Task 2: Conceptual Background

Interrupt-Driven ADC Conversion

  • Task 1 Approach: Polled ADC - CPU waits for each conversion to complete.
  • Task 2 Approach: Interrupt-driven ADC - CPU is notified when conversion is complete.
  • Advantage: CPU can perform other tasks while ADC conversion is in progress.
  • Use Case: More efficient for applications with multiple concurrent operations.

Task 2: Wiring & Key Functions

Additional Wiring

LDR (Voltage Divider - Dark Sensor Config): * 10kΩ Resistor → 3.3V * LDR → GND * Junction (between Resistor and LDR) → PA1 (ADC1_IN1)

LED (PWM Output): * LED anode → PA7 (TIM3_CH2) via 220Ω * LED cathode → GND

Key HAL Functions (Interrupt Mode)

  • HAL_ADC_Start_IT(): Start ADC with interrupt
  • HAL_ADC_ConvCpltCallback(): Auto-called when done
  • HAL_NVIC_EnableIRQ(): Enable interrupt
  • __HAL_TIM_SET_COMPARE(): Set LED brightness

Task 2: Code Implementation (Callback)

Step 5: The Interrupt Callback

In USER CODE BEGIN 4: Add callback:

void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef* hadc) {
  if (hadc->Instance == ADC1) {
    uint16_t ldr_value = HAL_ADC_GetValue(&hadc1);
    
    // Control LED Brightness
    // We want Bright LED in Dark -> High ADC directly maps to High Brightness.
    uint32_t led_brightness = (ldr_value * 999) / 4095;
    __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_2, led_brightness);
    
    // Send LDR Data to UART
    char msg[50];
    sprintf(msg, "LDR Value: %hu\r\n", ldr_value);
    HAL_UART_Transmit(&huart2, (uint8_t*)msg, strlen(msg), 100);

    HAL_ADC_Start_IT(&hadc1); // Restart for continuous operation
  }
}

Main loop: Can stay empty - interrupts handle everything!

Debugging & Common Pitfalls

Task 1 & Task 2 Troubleshooting

  • Task 1 - Servo Not Moving: Check 50Hz PWM configuration, verify pulse width range (500-2500).
  • Task 1 - Jittery Servo: Add a small delay between updates, check power supply stability.
  • Task 2 - Interrupt Not Firing: Verify NVIC interrupt is enabled in CubeMX.
  • Task 2 - LED Not Responding: Check callback function is properly defined, verify ADC channel configuration.
  • Both Tasks - ADC Reads Zero: Ensure GPIO pin is set to Analog mode, check voltage divider connections.
  • Both Tasks - PWM Not Working: Verify timer clock is enabled, check prescaler and ARR values.
