The long-main() problem

int main(void) {
    /* configure hardware */
    /* acquire eight samples */
    /* reject invalid values */

If every job lives in one function, changing one job can accidentally affect another.

Our first move is to put each job in a named function.

Reading a function

int clamp(int value, int low, int high)
{
    if (value < low) {
        return low;
    }
    if (value > high) {
        return high;
    }
    return value;
}

Inputs: value, low, high

Output: bounded value

Changes outside the function? No

Boundary cases: below, inside, above

Declare, define, call

1 · DECLARE

Tell other files the name and types

sensor_math.h

int clamp(int value,
          int low,
          int high);
  • Has no function body
  • Lets callers know the required types

→

2 · DEFINE

Write the function body

sensor_math.c

int clamp(int value,
          int low, int high)
{
    if (value < low)  return low;
    if (value > high) return high;
    return value;
}
  • Supplies the executable body
  • Normally exists once in the program

→

3 · CALL

Use the function

main.c

int safe = clamp(raw, 0, 4095);
  • Arguments must match the declaration
  • The return value replaces the call

COMPILE · The declaration lets the compiler check argument and return types.

LINK · The linker must find one matching definition of clamp.

Pass by value

void raise_to_minimum(int value)
{
    if (value < 100) {
        value = 100;
    }
}

int raw = 42;
raise_to_minimum(raw);

What changes?

The parameter value is a local copy.

raw remains 42.

Warning

Ordinary C parameters are passed by value. A return value sends one value back explicitly.

Deep dive: C11 draft N1570, §6.5.2.2 Function calls — p. 99.

Example: clamp()

int clamp(int value, int low, int high)
{

Common function mistakes

01 MISMATCH The declaration and definition use different types.

02 MISSING RETURN A non-void function reaches the end without returning a value.

03 WRONG ARGUMENTS A call uses the wrong type, order, or number of arguments.

Compiler warnings are evidence. Read them before debugging logic.

#include and headers

#include "sensor_math.h" is a preprocessor directive, not a function call. Before C compilation, a useful mental model is: replace that line with the header’s processed contents.

sensor_math.h

int clamp(int value,
          int low,
          int high);

sensor_math.c

#include "sensor_math.h"

int clamp(int value,
          int low,
          int high)
{
    /* implementation */
}

main.c

#include "sensor_math.h"

int main(void)
{
    int safe =
        clamp(raw, 0, 4095);
}

Both .c files now see the same declaration. The header publishes it; sensor_math.c defines it.

Deep dive: C11 draft N1570, §6.2.2 Linkages of identifiers — p. 54; #define and #include are §6.10 Preprocessing directives — p. 184.

Choosing integer widths

uint16_t sample = 4095U;
uint32_t sum = 0U;

sum = sum + sample;

sample stores one non-negative 16-bit reading.

sum may store the total of many readings, so it gets 32 bits.

The final average can return to uint16_t after division.

Choose a type for the largest value the variable may hold—not only the size of one input.

#define constants

#define SAMPLE_COUNT 8

uint16_t samples[SAMPLE_COUNT];

First: the preprocessor replaces every SAMPLE_COUNT with 8.

Then: the compiler sees uint16_t samples[8];.

Important: SAMPLE_COUNT is not a variable and does not use memory.

Use one readable name so the array size and loop count cannot accidentally use different numbers.

Arrays as parameters

uint16_t average_samples(
    uint16_t values[], int count);

uint16_t mean =
    average_samples(samples, 8);

values[] receives access to the first array element.

count receives 8, the number of valid elements.

Return value replaces the call and is stored in mean.

Warning

The function cannot discover the array length from values[]; the caller must also pass the count.

Example: average_samples()

uint16_t average_samples(uint16_t values[], int count)
{
    if (count == 0) {
        return 0U;
    }

    uint32_t sum = 0U;
    for (int i = 0; i < count; ++i) {
        sum += values[i];
    }

    return sum / count;
}

Use a 32-bit sum, handle count == 0, loop with i < count, and divide once at the end.

C strings

char status[8] = "READY";

0R

1E

2A

3D

4Y

5'\0'

6—

7—

The array capacity is 8; the text length is 5; the terminator occupies one element.

The \0 terminator

VALID STRING

{'O', 'K', '\0'}

Library functions know where to stop.

NOT A C STRING

{'O', 'K'}

No terminator is available.

The array size limits where we may write; \0 tells string code where the text stops.

Walking a string

int text_length(char text[])
{
    int i = 0;

    while (text[i] != '\0') {
        ++i;
    }

    return i;
}

Read one character at a time

Stop before counting \0

Return the number of characters

This loop works only if the array contains a \0 before its valid storage ends.

Comparing strings: strcmp()

DO NOT USE ==

if (command == "START") {
    /* ... */
}

This does not compare the characters.

USE strcmp()

if (strcmp(command, "START") == 0) {
    /* ... */
}

Zero means every character matched.

Include <string.h> and remember: strcmp(...) == 0 means equal.

Deep dive: C11 draft N1570, §7.24 String handling <string.h> — p. 380.
