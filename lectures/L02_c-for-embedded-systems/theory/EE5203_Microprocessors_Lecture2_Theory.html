Decisions and repetition

Last week, you built, flashed, and debugged a small C program. Today you will learn the C statements that make the program calculate, decide, and repeat.

int temperature = 24;
int fan_on = 0;

if (temperature > 22) {
    fan_on = 1;
}

Start by reading the code. Then predict the result before running it.

Today’s objectives

01 Store and change sensor values with variables.

02 Calculate with arithmetic operators.

03 Choose a result with if, else if, and else.

04 Repeat code with for and while.

05 Process all eight values in one array.

The laboratory keeps everything in one main.c file.

Statement order

int count = 0;
count = count + 1;
count = count + 1;

01 Create count and give it the value 0.

02 Read the current value, add 1, and store the result.

03 Do the same operation again.

After the third statement, count is 2.

Anatomy of a variable

int temperature = 24;

01 int stores a whole number.

02 temperature is the name used by later statements.

03 24 is the first value stored in the variable.

04 ; ends the statement.

Assignment

int temperature = 24;

temperature = 18;

The second statement does not ask a question. It stores 18 in temperature.

BEFORE

temperature is 24

AFTER

temperature is 18

Arithmetic operators

int a = 10;
int b = 3;

int sum        = a + b;   // 13
int difference = a - b;   // 7
int product    = a * b;   // 30
int quotient   = a / b;   // 3
int remainder  = a % b;   // 1

Both operands of a / b are integers, so the result is the whole-number quotient.

++ and +=

int count = 0;
int sum = 10;

count++;     // same result as count = count + 1
sum += 5;    // same result as sum = sum + 5

Use the longer form first when it makes the operation easier to understand.

Note

After these statements, count is 1 and sum is 15.

Practice: predict the values

int x = 4;
int y = 2;

x = x + 3;
y *= 5;
x++;

1 What is the final value of x?

2 What is the final value of y?

3 Which line would you execute first in the debugger?

Comparison operators

Expression Question
a == b Are the values equal?
a != b Are the values different?
a < b Is a smaller?
a <= b Is a smaller or equal?
a > b Is a larger?
a >= b Is a larger or equal?

A comparison does not change either variable.

Deep dive: C11 draft N1570, §6.8.4 Selection statements — p. 166 covers if; comparisons are §6.5.8–6.5.9.

= vs ==

ASSIGNMENT

status = 1;

Store 1 in status.

COMPARISON

status == 1

Ask whether status contains 1.

Using = when you mean == is a common beginner error.

The if statement

int temperature = 24;
int fan_on = 0;

if (temperature > 22) {
    fan_on = 1;
}

01 Test temperature > 22.

02 If it is true, execute the statement inside the braces.

03 If it is false, skip that statement.

The if/else decision

if (temperature > 22) {
    fan_on = 1;
} else {
    fan_on = 0;
}

Exactly one block runs.

For temperature = 24, the program stores 1. For temperature = 18, it stores 0.

else if chains

if (sample < 1500) {
    status_code = -1;       // LOW
} else if (sample > 3000) {
    status_code = 1;        // HIGH
} else {
    status_code = 0;        // OK
}

The program tests from top to bottom and stops at the first true condition.

Logical operators

Operator Meaning Example
&& both conditions must be true sample >= 0 && sample <= 4095
|| at least one condition is true mode == 1 || mode == 2
! reverse true and false !ready

Use parentheses when they make the intended grouping clearer.

Practice: classify one sample

int sample = 2800;
int status_code = 0;

if (sample < 1500) {
    status_code = -1;
} else if (sample > 3000) {
    status_code = 1;
} else {
    status_code = 0;

1 Which condition is tested first?

2 What is the final value of status_code?

3 What changes when sample is 3500?

The while loop

int count = 0;

while (count < 3) {
    count++;
}

The loop tests count < 3 before every repetition.

The body runs with count equal to 0, 1, and 2. The loop stops when count becomes 3.

The for loop

for (int i = 0; i < 4; i++) {
    /* use i */
}

01 START: create i with value 0.

02 TEST: continue while i < 4.

03 UPDATE: increase i after each repetition.

Loop trace table

Repetition i before body Is i < 4 true?
1 0 yes
2 1 yes
3 2 yes
4 3 yes
stop 4 no

The loop body runs four times, not five.

Deep dive: C11 draft N1570, §6.8.5 Iteration statements — p. 168.

Which statement when?

01 Use if when code may run once or may be skipped.

02 Use while when repetition depends on a condition.

03 Use for when a counter visits a known range.

04 Use while (1) for the main loop of firmware that must keep running.

The forever loop

int main(void)
{
    /* initialization runs once */

    while (1) {
        /* repeated work */
    }
}

The startup code calls main(). Initialization runs first; the infinite loop then keeps the firmware alive.

The uint16_t type

#include <stdint.h>

uint16_t sample = 4095;

01 uint means unsigned integer: no negative values.

02 16 means the storage has 16 bits.

03 _t is part of the standard type name.

Use ordinary int for small counters and calculations in this week’s program.

Deep dive: C11 draft N1570, §7.20 Integer types <stdint.h> — p. 307.

Arrays

uint16_t samples[8] = {
    850, 1250, 4095, 4200,
    2050, 900, 3100, 2800
};
  • uint16_t is the type of every element.
  • samples is the array name.
  • [8] reserves space for eight elements.

Array indexing

Expression Selected element
samples[0] first
samples[1] second
samples[7] eighth
samples[8] invalid

Looping over an array

for (int i = 0; i < 8; i++) {
    samples[i] = samples[i] + 1;
}

During each repetition, i selects one element.

The condition is i < 8, not i <= 8, because index 8 is invalid.

The accumulator pattern

int sum = 0;

for (int i = 0; i < 8; i++) {
    sum += samples[i];
}

int average = sum / 8;

sum must start at zero. Each repetition adds one element.

One loop, several results

int sum = 0;
int high_count = 0;

for (int i = 0; i < 8; i++) {
    if (samples[i] > 4095) {
        samples[i] = 4095;
    }

    sum += samples[i];

    if (samples[i] > 2500) {
        high_count++;
    }
}

This loop clamps invalid values, calculates a sum, and counts high samples.

The complete Week 2 program

uint16_t samples[8] = {
    850, 1250, 4095, 4200,
    2050, 900, 3100, 2800
};

int sum = 0;
int high_count = 0;

for (int i = 0; i < 8; i++) {
    if (samples[i] > 4095) {
        samples[i] = 4095;
    }
    sum += samples[i];
    if (samples[i] > 2500) {
        high_count++;
    }
}

int average = sum / 8;

Predict, then verify

01 Calculate the expected final array, sum, average, and high count.

02 Build with zero errors and read the first warning, if any.

03 Break after the loop and inspect the variables.

04 Change one input value, predict again, and repeat the test.

Common Week 2 mistakes

01 Using = instead of == in a condition.

02 Forgetting to give an accumulator its starting value.

03 Writing i <= 8 and accessing samples[8].

04 Forgetting to update the counter in a while loop.

Laboratory mission

Process eight simulated sensor values in one main.c file.

You will:

  1. clamp values above 4095;
  2. calculate the sum and average;
  3. count values above 2500;
  4. select a numeric LOW, OK, or HIGH status;
  5. prove the result in the debugger.

Exit ticket

int values[4] = {3, 7, 2, 8};
int total = 0;

for (int i = 0; i < 4; i++) {
    if (values[i] > 4) {
        total += values[i];
