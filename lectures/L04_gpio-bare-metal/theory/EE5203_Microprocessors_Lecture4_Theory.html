Memory addresses

Address Example stored value
0x20000000 24
0x20000004 1
0x20000008 4095

The value is the data. The address identifies where that data is stored.

Addresses are commonly written in hexadecimal because it is shorter than writing all the address bits.

Where these addresses live—flash, SRAM, and peripherals on one 32-bit map: From C to Silicon, §2 The memory your program lives in.

Taking an address: &

int temperature = 24;
int *address = &temperature;

01 temperature produces the stored value: 24.

02 &temperature produces the address of that variable.

03 int *address declares a pointer that can hold the address of an int.

Read the declaration aloud: “address is a pointer to an int.”

Following a pointer: *

int temperature = 24;
int *address = &temperature;

*address = 18;

The final statement follows the stored address and writes 18 there.

After the write, temperature and *address both produce the value 18.

Deep dive: C11 draft N1570, §6.5.3.2 Address and indirection operators — p. 106.

Practice: pointer trace

int x = 5;
int *p = &x;

*p = *p + 3;

1 Which variable’s address is stored in p?

2 What value does *p produce before the assignment?

3 What is the final value of x?

p stores &x; *p first produces 5; the assignment stores 8 in x.

Structures

struct Point {
    int x;
    int y;
};

struct Point target = {4, 7};

x and y are members of the same struct Point value.

A peripheral definition uses the same idea: one structure groups all registers belonging to that peripheral.

Deep dive: C11 draft N1570, §6.7.2.1 Structure and union specifiers — p. 130.

Member access: . and ->

STRUCTURE VALUE

struct Point target;

target.x = 4;

Read: “the x member of target.”

POINTER TO A STRUCTURE

struct Point *p = &target;

p->x = 4;

Read: “the x member reached through p.”

p->x is a shorter form of (*p).x.

Reading GPIOA->MODER

GPIOA->MODER

01 GPIOA is a CMSIS-provided pointer to the GPIOA register structure.

02 -> selects a member through that pointer.

03 MODER is the 32-bit mode register inside GPIOA.

The device header supplies this definition; your program uses the readable name.

The 32-bit register

Each bit can be 0 or 1. The reference manual assigns a meaning to each bit or group of bits.

uint32_t register_value = 0U;

uint32_t matches the width of an STM32 peripheral register.

Hexadecimal

Hex Binary Decimal
0 0000 0
5 0101 5
A 1010 10
F 1111 15 
0x00000020 = 0000 0000 0000 0000 0000 0000 0010 0000

A 32-bit register can be written with eight hexadecimal digits.

Bitwise operators

Operator Direct meaning
& keep a bit only when both input bits are 1
| make a bit 1 when either input bit is 1
^ make the result 1 when the input bits differ
~ reverse every bit
<< shift bits left
>> shift bits right

These are different from the logical operators &&, ||, and ! used in conditions.

Deep dive: C11 draft N1570, §6.5.7 Bitwise shift operators — p. 112 (then §6.5.10–6.5.12 for &, ^, |).

Building a mask

uint32_t mask = (1U << 5);
1U          0000 0001
1U << 5     0010 0000

The result has only bit 5 set. This is called a mask because it selects a specific bit position.

The U suffix marks the number as unsigned.

Set a bit: OR

register_value |= (1U << 5);
old value     0000 0001
mask          0010 0000
OR result     0010 0001

Bit 5 becomes 1; the other bits keep their previous values.

Clear a bit: AND + NOT

register_value &= ~(1U << 5);

01 1U << 5 selects bit 5.

02 ~ changes that mask so bit 5 is 0 and the other bits are 1.

03 &= clears bit 5 and preserves the other bits.

Replacing a bit field

Suppose bits 11:10 must become 01.

register_value &= ~(3U << 10);  // clear bits 11:10
register_value |=  (1U << 10);  // write 01

Clear the whole field first. Then set only the bits required by the new pattern.

3U is binary 11, so 3U << 10 selects both field bits.

Toggle and test

TOGGLE BIT 5

register_value ^= (1U << 5);

Zero becomes one; one becomes zero.

TEST BIT 5

if ((register_value & (1U << 5)) != 0U) {
    bit_is_set = 1;
}

A nonzero result means bit 5 was set.

CMSIS register names

REFERENCE MANUAL

Peripheral Register Purpose
RCC AHB1ENR enable GPIO clocks
GPIOA MODER choose each pin’s mode
GPIOA BSRR set or reset output pins

C CODE

RCC->AHB1ENR
GPIOA->MODER
GPIOA->BSRR

The selected STM32 device header supplies these names and their addresses.

This is direct register access: CMSIS provides the names, but your C statement still reads or writes the hardware register.

The definitions live in stm32f401xe.h — every generated STM32 project includes this file.

volatile registers

typedef struct
{
    volatile uint32_t MODER;
    /* other GPIO registers */
    volatile uint32_t BSRR;
} GPIO_TypeDef;

Hardware can change a register outside normal C program flow. volatile tells the compiler that every register read and write must actually happen.

Use volatile for hardware-controlled data—not for every ordinary variable.

Deep dive: C11 draft N1570, §6.7.3 — p. 140 · M. Barr, How to Use C’s volatile Keyword.

The GPIOA clock

RCC->AHB1ENR |= (1U << 0);

Bit 0 of AHB1ENR enables the clock for GPIOA.

01 RCC selects the reset-and-clock-control peripheral.

02 AHB1ENR selects its AHB1 peripheral clock-enable register.

03 |= (1U << 0) sets the GPIOA enable bit without changing other clock bits.

Enable the clock first—an unclocked peripheral ignores register writes.

Deep dive: RM0368 Rev 6, RCC_AHB1ENR — p. 118.

PA5’s mode field

Every GPIO pin uses two mode bits. Pin 5 therefore begins at bit 5 × 2 = 10.

Bits 11:10 PA5 mode
00 input
01 general-purpose output
10 alternate function
11 analog

PA5 as output

GPIOA->MODER &= ~(3U << 10);
GPIOA->MODER |=  (1U << 10);

01 3U << 10 selects bits 11 and 10.

02 The first statement clears both bits to 00.

03 The second statement sets bit 10, producing 01.

Do not assume the old field value; clear before setting the new pattern.

Deep dive: RM0368 Rev 6, GPIOx_MODER — p. 158.

Driving PA5 with BSRR

GPIOA->BSRR = (1U << 5);       // PA5 high: LD2 on
GPIOA->BSRR = (1U << 21);      // PA5 low:  LD2 off

Direct assignment is correct because each BSRR write is a command, not a stored configuration field.

Deep dive: RM0368 Rev 6, GPIOx_BSRR — p. 161.

The bare-metal blink

RCC->AHB1ENR |= (1U << 0);

GPIOA->MODER &= ~(3U << 10);
GPIOA->MODER |=  (1U << 10);

while (1) {
    GPIOA->BSRR = (1U << 5);
    HAL_Delay(250);

    GPIOA->BSRR = (1U << 21);
    HAL_Delay(250);
}

Configuration happens once before the loop; only the BSRR writes repeat.

The delay still uses a HAL service. GPIO configuration and output control use direct CMSIS register writes.

Debugging register writes

01 PREDICT: calculate which bit or field should change.

02 STEP: stop immediately before and after the statement.

03 INSPECT: read the register in the SFR/peripheral view.

04 COMPARE: match the observed hexadecimal value to the mask.

05 CONFIRM: check that the physical LED agrees with the register state.

Register maps for the SFR view: RM0368 Rev 6, §8 GPIO — p. 146.

Common register mistakes

01 WRONG DEVICE: the project targets a different STM32 model.

02 CLOCK OFF: GPIOA is used before its clock-enable bit is set.

03 WRONG SHIFT: the mask selects a different pin or field.

04 FIELD NOT CLEARED: old mode bits remain in the two-bit field.

05 WRONG EVIDENCE: a successful build is treated as proof that the hardware changed.
